## 踩坑记录2：流式输出被吞了一行

项目需要实时显示回复，所以我改成了流式调用：

javascript
const stream = await client.chat.completions.create({
  model: 'qwen-turbo',
  messages: [...],
  stream: true,
});
for await (const chunk of stream) {
  process.stdout.write(chunk.choices[0]?.delta?.content || '');
}


这段代码本身没毛病，但第一次跑发现最后一段完整内容经常被截掉一点。后来发现是终端输出缓冲导致的，和API无关。把 `process.stdout.write` 换成 `console.log` 就没这问题了。

**教训：** 流式调用一定要本地先测试好输出，别把锅甩给API。

## 踩坑记录3：环境变量没配，硬编码Key被传上Git

这是很多新手犯的错误。你把API Key直接写在 `apiKey: 'xxxx'` 里，然后 `git push` 到公共仓库。那次我差点把公司Key泄露出去。

**正确做法**：使用 `.env` 文件

安装 `dotenv`：
bash
npm install dotenv


然后在项目根目录新建 `.env` 文件：

DEEPSEEK_API_KEY=你的云雾API Key


最后在代码里这样调用：
javascript
require('dotenv').config();
const client = new OpenAI({
  apiKey: process.env.DEEPSEEK_API_KEY,
  baseURL: 'https://www.yunwuai.cc/v1'
});


记得在 `.gitignore` 里加上 `.env`。这一步很关键，安全永远排第一。
