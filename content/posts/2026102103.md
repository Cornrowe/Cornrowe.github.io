### 2. 核心调用代码（复制即用）

编写一个核心服务类 `Gpt51Service.java`，用于封装所有 API 调用逻辑。以下是关键部分的代码，**只需替换三处**即可运行。

java
import com.theokanning.openai.service.OpenAiService;
import com.theokanning.openai.completion.chat.ChatCompletionRequest;
import com.theokanning.openai.completion.chat.ChatMessage;
import java.time.Duration;
import java.util.List;

public class Gpt51Service {

    // 【第1处替换】换成你在云雾平台创建的 API Key
    private static final String API_KEY = "sk-your-yunwuai-api-key-here";

    // 【第2处替换】设置云雾平台的国内直连端点
    private static final String BASE_URL = "https://www.yunwuai.cc/v1";

    // 【第3处替换】指定模型名称为 GPT-5.1 最新版
    private static final String MODEL_NAME = "gpt-5.1";

    private final OpenAiService service;

    public Gpt51Service() {
        // 构建 client 时传入自定义 baseUrl
        OkHttpClient client = new OkHttpClient.Builder()
                .baseUrl(BASE_URL)
                .apiKey(API_KEY)
                .connectTimeout(60, TimeUnit.SECONDS)
                .build();
        this.service = new OpenAiService(client);
    }

    /**
     * 发送对话请求，并获得流式回复（企业级应用推荐）
     */
    public String chat(String userMessage) {
        List<ChatMessage> messages = List.of(
                new ChatMessage("user", userMessage)
        );

        ChatCompletionRequest completionRequest = ChatCompletionRequest.builder()
                .model(MODEL_NAME)
                .messages(messages)
                .maxTokens(4096)
                .temperature(0.7)
                .build();

        // 流式输出
        StringBuilder response = new StringBuilder();
        service.streamChatCompletion(completionRequest)
                .doOnError(Throwable::printStackTrace)
                .blockingForEach(chunk -> {
                    if (chunk.getChoices() != null && !chunk.getChoices().isEmpty()) {
                        response.append(chunk.getChoices().get(0).getMessage().getContent());
                    }
                });
        return response.toString();
    }

    public static void main(String[] args) {
        Gpt51Service gpt = new Gpt51Service();
        String reply = gpt.chat("请用中文介绍一下GPT-5.1相比GPT-4的改进点。");
        System.out.println("GPT-5.1 回复：" + reply);
    }
}


**说明：**
- 使用流式输出模式，适合企业级应用中需要实时展示生成内容（如客服、编辑器辅助）。
- 最大 tokens 设为 4096，可根据业务需要调整。

## 复杂企业场景：多轮对话 + 上下文记忆

在实际企业应用中，我们往往需要聊天机器人或智能助手记住上下文。下面给出一个改进版，支持**多轮对话记录**。

java
public class Gpt51MemoryChat {
    private final Gpt51Service service;
    private final List<ChatMessage> conversationHistory = new ArrayList<>();

    public Gpt51MemoryChat() {
        this.service = new Gpt51Service();
    }

    public String chatWithMemory(String userInput) {
        conversationHistory.add(new ChatMessage("user", userInput));

        ChatCompletionRequest request = ChatCompletionRequest.builder()
                .model("gpt-5.1")
                .messages(conversationHistory)
                .maxTokens(4096)
                .temperature(0.7)
                .build();

        StringBuilder fullResponse = new StringBuilder();
        service.streamChatCompletion(request)
                .blockingForEach(chunk -> {
                    if (chunk.getChoices() != null && !chunk.getChoices().isEmpty()) {
                        fullResponse.append(chunk.getChoices().get(0).getMessage().getContent());
                    }
                });

        conversationHistory.add(new ChatMessage("assistant", fullResponse.toString()));
        return fullResponse.toString();
    }
}


这种写法在智能客服、代码助手、文档问答等场景下非常实用。
