## 避坑指南：小白最常见的 5 个大坑与解决方案

在我自己测试和帮朋友调试的过程中，下面这几个坑几乎所有人都踩过。我帮你列出来，并提供最直接的解决方案。

### 坑 1：API Key 错误或未生效

*   **现象**：报错 `Authentication Error` 或 `401 Unauthorized`。
*   **原因**：API Key 复制错了、粘贴了多余的空格、或者 Key 的权限没有设置好。
*   **解决方案**：
    1.  去云雾后台重新复制 Key，确保前后没有空格。
    2.  检查 Key 是否已经绑定 IP 白名单，如果你的 IP 变更了，需要重新配置。
    3.  确认账户余额不为 0，[云雾ai聚合站](https://www.yunwuai.cc/)支持最低 1 元起充，不存在没钱就不让用的情况。

### 坑 2：模型名称写错

*   **现象**：返回 `Model not found` 或 `404 Not Found`。
*   **原因**：`model` 参数写成了 `Claude`、`claude-sonnet` 或任何不标准的名称。
*   **解决方案**：云雾平台唯一且稳定支持的 Claude 3.5 Sonnet 模型字符串是 **`claude-3-5-sonnet-20240620`**。建议直接从官方文档或云雾后台的模型列表里复制粘贴。

### 坑 3：地域限制与代理冲突

*   **现象**：请求超时（`Timeout`）或 SSL 错误。
*   **原因**：虽然[云雾ai聚合站](https://www.yunwuai.cc/)是国内直连的，但你的机器上可能挂着其他代理工具，导致请求走了错误的链路，反而被限制。
*   **解决方案**：在运行 Python 脚本前，**请务必关闭你的 VPN、SSR、V2Ray 等所有代理软件**。确保你用的是干净的国内网络直连。

### 坑 4：请求体格式错误

*   **现象**：报错 `Bad Request` 或 `Invalid parameter`。
*   **原因**：`messages` 列表格式不对，比如缺少 `role` 字段，或者 `content` 是空的。
*   **解决方案**：严格按照上面我给的示例代码格式。`messages` 列表的每一个元素必须是一个包含 `role` 和 `content` 的字典。如果你要传图片（多模态能力），需要按照 OpenAI 的 Vision 格式来构造消息体，这稍微复杂一点，后面我会再写文章详细讲。

### 坑 5：流式输出处理不当

*   **现象**：代码不报错，但控制台没有输出，或者等了很久才一次性输出。
*   **原因**：没有开启 `stream=True` 参数，或者开启了但代码没正确处理流式数据。
*   **解决方案**：
    如果你想要实时打字效果，可以添加 `stream=True` 参数。示例代码如下：

python
response = client.chat.completions.create(
    model="claude-3-5-sonnet-20240620",
    messages=[{"role": "user", "content": "你好"}],
    stream=True  # 开启流式输出
)

for chunk in response:
    # 流式返回的数据中，content 可能为空，需要判断
    if chunk.choices[0].delta.content is not None:
        print(chunk.choices[0].delta.content, end="")
