来看核心的调用代码，非常简单，就几十行。

java
import okhttp3.*;
import com.google.gson.Gson;
import com.google.gson.JsonObject;
import java.io.IOException;

public class GrokJavaExample {
    private static final String API_BASE_URL = "https://www.yunwuai.cc/v1";
    private static final String API_KEY = "sk-你的云雾API Key"; // 别直接放代码里！建议配到环境变量
    private static final String MODEL = "grok-1"; // 具体模型名以云雾后台为准

    public static void main(String[] args) throws IOException {
        OkHttpClient client = new OkHttpClient().newBuilder().build();

        // 构建请求体
        JsonObject requestBody = new JsonObject();
        requestBody.addProperty("model", MODEL);
        // 构建消息
        JsonObject userMessage = new JsonObject();
        userMessage.addProperty("role", "user");
        userMessage.addProperty("content", "用中文简单地介绍一下你自己是谁");
        // 注意：复杂请求可能需要构建消息数组，这里只放一个用户消息
        // 实际使用需构建完整的messages数组
        // 为简化演示，这里模型会视为一次对话

        // Grok的标准请求格式，message数组是必须的
        // 我们手动构建一个包含用户消息的数组
        String jsonString = """
                {
                    "model": "%s",
                    "messages": [
                        {"role": "user", "content": "用中文简单地介绍一下你自己是谁"}
                    ]
                }
                """.formatted(MODEL);

        MediaType mediaType = MediaType.parse("application/json");
        RequestBody body = RequestBody.create(mediaType, jsonString);

        Request request = new Request.Builder()
                .url(API_BASE_URL + "/chat/completions")
                .method("POST", body)
                .addHeader("Authorization", "Bearer " + API_KEY)
                .addHeader("Content-Type", "application/json")
                .build();

        try (Response response = client.newCall(request).execute()) {
            if (response.isSuccessful()) {
                String responseString = response.body().string();
                System.out.println("Grok Says:\n" + responseString);
                // 你可以用Gson解析responseString，提取content字段
                JsonObject jsonResponse = new Gson().fromJson(responseString, JsonObject.class);
                String answer = jsonResponse.getAsJsonArray("choices")
                        .get(0).getAsJsonObject()
                        .getAsJsonObject("message")
                        .get("content").getAsString();
                System.out.println("模型回答内容:\n" + answer);
            } else {
                System.out.println("请求失败，状态码: " + response.code());
                System.out.println("错误信息: " + response.body().string());
            }
        }
    }
}


你看，复杂性完全被封装了。你不需要花里胡哨的依赖，几行OkHttp + Gson就能完成核心逻辑。[云雾api聚合站](https://www.yunwuai.cc/)的接口完全兼容OpenAI的 `chat/completions` 端点，所以你的学习曲线非常平滑，换模型基本只是改一个 `MODEL` 字符串的事。
