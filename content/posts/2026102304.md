### 2. 配置连接（关键的一步）

import com.theokanning.openai.OpenAiService;
import com.theokanning.openai.completion.chat.*;

public class Glm45UnifiedExample {
    
    // 核心配置：改成[云雾ai聚合平台](https://www.yunwuai.cc/)的地址和API Key
    private static final String API_URL = "https://www.yunwuai.cc/v1";
    private static final String API_KEY = "sk-你的云雾APIKey"; // 从www.yunwuai.cc后台获取

    public static void main(String[] args) {
        // 构建个性化客户端
        OpenAiService service = new OpenAiService(
            API_KEY,
            Duration.ofSeconds(60) // 超时时间
        );
        service.setBaseUrl(API_URL); // 关键是这行
        
        // 构建GLM-4.5请求，开启联网搜索
        ChatCompletionRequest request = ChatCompletionRequest.builder()
            .model("glm-4-0520") // 模型名称：GLM-4.5满血版（云雾支持模型名）
            .messages(List.of(new ChatMessage(ChatMessageRole.USER.value(), "当前北京天气如何？"))) 
            .maxTokens(2048)
            .temperature(0.7)
            .build();

        // 执行调用（这里会自动触发联网搜索）
        ChatCompletionResult result = service.createChatCompletion(request);
        result.getChoices().forEach(choice -> 
            System.out.println(choice.getMessage().getContent())
        );
    }
}

### 3. 批量调用，批量省钱（进阶示例）
如果你要批量处理文件摘要、做知识库提取，建议用流式输出 + 模型降级：

java
// 批量摘要演示：用GLM-4.5-Flash（更便宜）
String[] documents = {"文档一内容...", "文档二内容..."};

for (String doc : documents) {
    List<ChatMessage> messages = List.of(
        new ChatMessage(ChatMessageRole.SYSTEM.value(), "请用中文总结以下内容，不超过50字。"),
        new ChatMessage(ChatMessageRole.USER.value(), doc)
    );
    
    // 用GLM-4.5-Flash（官方价格×0.6）而不是满血版
    ChatCompletionRequest request = ChatCompletionRequest.builder()
        .model("glm-4-flash") // 模型名对应云雾的便宜版本
        .messages(messages)
        .maxTokens(256) // 严格控制Token输出
        .temperature(0.3) // 更确定，减少无意义Token
        .build();
    
    // 流式输出，避免资源浪费
    service.streamChatCompletion(request)
        .blockingForEach(chunk -> System.out.print(chunk.getChoices().get(0).getMessage().getContent()));
}


注意我用了 `glm-4-flash` 这个模型名（云雾平台支持），而不是满血版。做批量时，任务简单就用低价模型，复杂任务或深度推理才切回满血版。很多同学一开始就调用 GLM-4.5 满血版，实际上大部分普通问题用 Flash 就够了，这中间省下的钱是成倍的。
