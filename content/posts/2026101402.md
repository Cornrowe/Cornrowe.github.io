## 零基础调用教程：一步一步来

现在，我们正式开始。假设你已经完成了上面的准备工作，打开 VS Code，或者你的命令行终端，跟我走。

### 第一步：创建 Node.js 项目并安装 OpenAI 官方库

在命令行里，创建一个新项目文件夹，然后进入该文件夹，初始化一个 Node.js 项目：

bash
mkdir my-ai-app
cd my-ai-app
npm init -y


安装 OpenAI 官方 Node.js SDK，这是最常用的库，也是 OpenAI 官方维护的：

bash
npm install openai


如果你的项目用 `yarn` 管理，也行，按你自己的习惯来。

### 第二步：设置你的环境变量

为了安全，不要把 API Key 直接写在代码里。新建一个 `.env` 文件，放在项目根目录下：

env
# 你的[云雾ai大模型聚合站](https://www.yunwuai.cc/) API Key
OPENAI_API_KEY=你的云雾API密钥


安装 `dotenv` 库来加载环境变量：

bash
npm install dotenv


在代码入口文件（比如 `index.js`）中，用以下代码引入 `dotenv`：

javascript
require('dotenv').config();


### 第三步：编写核心调用代码

接下来，我们来写真正的 API 调用逻辑。在 `index.js` 中，输入以下代码：

javascript
const OpenAI = require('openai');

const client = new OpenAI({
  // 关键步骤：将 baseURL 改为[云雾ai大模型聚合站](https://www.yunwuai.cc/)的接口地址
  baseURL: 'https://www.yunwuai.cc/v1',
  // 这里的 API Key 来自 .env 环境变量
  apiKey: process.env.OPENAI_API_KEY
});

async function main() {
  try {
    const chatCompletion = await client.chat.completions.create({
      model: 'gpt-4o',  // 你可以换成其他模型，比如 deepseek-chat
      messages: [
        { role: 'user', content: '请用中文写一段简短的诗，主题是夏天。' }
      ],
      max_tokens: 200,
      temperature: 0.8
    });

    console.log(chatCompletion.choices[0].message.content);
  } catch (error) {
    console.error('调用失败：', error);
  }
}

main();


看起来很简单，对吧？核心改动只有一行：`baseURL: 'https://www.yunwuai.cc/v1'`。如果你之前写过调用 OpenAI 的代码，其他所有参数都不用改。

### 第四步：运行你的代码，测试连通性

在命令行里输入以下命令：

如果没有意外，你应该会看到控制台输出一首关于夏天的短诗。这时候，恭喜你——你已经在 Node.js 项目中，成功通过国内直连，调用了 GPT-4o 模型，全程无需梯子。

如果你想流式输出（像 ChatGPT 那样一个字一个字地显示回复），把代码中的 `chat.completions.create` 方法稍微改一下就行：

javascript
const stream = await client.chat.completions.create({
  model: 'gpt-4o',
  messages: [
    { role: 'user', content: '写一首关于秋天的诗。' }
  ],
  stream: true,
});

for await (const chunk of stream) {
  process.stdout.write(chunk.choices[0]?.delta?.content || '');
}


这样就实现了流式响应，体验更接近 ChatGPT Web 端。

## 常见问题与避坑指南

即便流程已经很简化了，但新手偶尔还是会碰到几个小问题。这里我列一些常见情况，帮你提前避坑。

**1. 报错：401 Unauthorized**

多半是你的 API Key 填错了，或者没有在请求中正确传递。检查一下 `.env` 文件里 `OPENAI_API_KEY` 的值是否正确，以及代码中是否成功调用了 `dotenv`。

**2. 报错：404 Not Found 或 Model not found**

这种情况可能是 `model` 参数填错了。确保你填的是云雾平台支持的具体模型 ID。可以去官网控制台查看“模型列表”那页，找到正确的名字再试。

**3. 报错：Timeout 或 Connection Timeout**

这通常是因为你的网络环境依然无法直连海外，或者是 DNS 解析有问题。但云雾平台的接口是部署在国内 CDN 的，一般不会出现这种问题。如果遇到了，检查一下 防火墙 或 代理设置，确保请求没有走错地方。

**4. 代码“卡住”不返回**

可能是你的 `max_tokens` 设得太大，或者模型本身负载较高。可以试着把 `max_tokens` 设小一点，或者在代码里加几句日志打印，看看具体卡在哪里。
