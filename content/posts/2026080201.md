## 手把手写代码：Node.js接入Qwen-Plus

好了，现在进入最核心的部分。我假设你已经有一个安装了Node.js的开发环境（版本16以上），如果没有，去官网下载一个就行，五分钟搞定。

### 安装openai NPM包

在项目目录下打开终端，执行：

bash
npm init -y
npm install openai


这个`openai`包是OpenAI官方发布的SDK，但它的设计是支持自定义`baseURL`的——这就是我们能用它来调用非OpenAI模型的关键。

### 创建主程序文件

在项目根目录下创建`demo.js`，输入以下代码：

javascript
import OpenAI from 'openai';

const openai = new OpenAI({
  baseURL: 'https://www.yunwuai.cc/v1',
  apiKey: 'YOUR_API_KEY_HERE', // 替换为你的Key
});

async function main() {
  const stream = await openai.chat.completions.create({
    model: 'Qwen-Plus',
    messages: [
      { role: 'system', content: '你是一个专业的AI助手，回答要简洁准确。' },
      { role: 'user', content: '用通俗的语言解释一下什么是量子计算？' },
    ],
    stream: true, // 开启流式输出，体验更丝滑
  });

  console.log('Qwen-Plus 回答：');
  for await (const chunk of stream) {
    const content = chunk.choices[0]?.delta?.content || '';
    process.stdout.write(content);
  }
  console.log('\n--- 对话结束 ---');
}

main().catch(console.error);


**关键点解释：**
- `baseURL`：必须设置为`https://www.yunwuai.cc/v1`，这是[云雾ai大模型聚合站](https://www.yunwuai.cc/)提供的统一接入点
- `model`：指定为`'Qwen-Plus'`，这是你的目标模型ID
- `stream: true`：开启流式输出，你可以像打字机一样看到内容逐字出现，用户反馈很好
- `apiKey`：粘贴刚才在控制台复制的Key

## 进阶技巧：非流式输出与错误处理

流式输出适合聊天交互，但如果你在做文本批处理或者需要完整结果再展示的场景，可以用非流式模式。代码只有一处不同：

javascript
const response = await openai.chat.completions.create({
  model: 'Qwen-Plus',
  messages: [ /* 同上 */ ],
  stream: false,
});

console.log(response.choices[0].message.content);


另外，实际开发中务必加错误处理，防止网络波动导致程序崩溃：

javascript

  const response = await openai.chat.completions.create({ /* 配置 */ });
  console.log(response.choices[0].message.content);
} catch (error) {
  if (error.status === 401) {
    console.error('API Key无效，请检查');
  } else if (error.status === 429) {
    console.error('请求频率过高，稍后重试');
  } else {
    console.error('其他错误:', error.message);
  }
}
