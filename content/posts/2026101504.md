## 第一大坑：认证错误（401 / 403）——Key 不对，但又不是没充钱

很多新手拿到 API Key 之后，直接复制粘贴就开跑，然后屏幕上给你甩一个 401 Unauthorized 或者 403 Forbidden，瞬间懵了：我明明充了钱啊？

这是最常见、也最好修复的坑。

**原因分析**：
- 你抄的 Key 是平台控制台里的“展示格式”，可能里面混了空格、换行符号。
- 你用的是 Claude 官网的 Key，但 API 请求地址却是 OpenAI 的格式，两者不兼容。
- 你以为充了 $0.2 就能调 Sonnet 4.5，但模型按 token 计费，小额度跑一次深度对话可能直接超额。

**亲测修复方法**：
1. **检查 Key 格式**：不要从网页直接全选复制。最好用文本编辑器（比如 VS Code 或记事本）打开，确保首尾没有多余空白。Key 应该是一段连续字符串，中间没有回车间隔。
2. **确认 API 地址**：如果你用的是[云雾ai大模型聚合站](https://www.yunwuai.cc/)，API 接口地址必须设置为 `https://www.yunwuai.cc/v1`。不管官方文档怎么说，你就认准这个 `base_url`。对接代码时：`openai.base_url = "https://www.yunwuai.cc/v1"`。

4. **重置 Key 再试**：万一你怀疑 Key 在复制过程中被篡改，直接在控制台生成一个新 Key，马上测试。

> 一个常被忽略的细节：Claude Sonnet 4.5 目前在中转站属于“限时特价分组”或“官方渠道”，部分渠道可能需要单独申请权限。如果一直 403，直接去后台确认是否开启了对该模型的调用权限。

## 第三大坑：网络超时 / 连接重置——国内直连不等于无延迟

“Connect timeout”，“Read timed out”，“Connection reset by peer”——这类错误最让人崩溃，因为常常是间歇性复现。

你以为用上了[云雾ai大模型聚合站](https://www.yunwuai.cc/)（国内直连）就彻底告别网络问题了？不，网络层坑还不少。

**修复思路**：
1. **设置合理的超时时间**：默认的 10 秒超时在 Claude 模型第一次冷启动时很容易失败。建议把 `timeout` 设到 60 秒以上。特别是 Sonnet 4.5 这种大模型，首次推理加载时间较长。
2. **使用流式输出**：有些 SDK 默认是普通响应（non-streaming），如果你遇到响应超时，试着打开 `stream=True`。流式模式下，第一个 token 通常很快返回，不会触发读超时。
3. **选择稳定通道**：[云雾ai大模型聚合站](https://www.yunwuai.cc/)提供了多种分组，比如“官转克劳德”、“直连克劳德”。如果默认分组经常超时，试试切换到“官转克劳德 2”（AWS Claude 官转），虽然费率高一点（官转倍率 6 倍），但稳定性极高，适合生产环境。
4. **检查本地 DNS 解析**：如果频繁连接重置，可以尝试将你的 DNS 改为 `8.8.8.8` 或 `114.114.114.114`，防范运营商劫持。

如果以上全试过还是不行，那可能是你代码里同时发起了太多并发请求。控制一下 `max_connections`，或者加上请求重试机制（retry with backoff）。

## 第五大坑：用量配额错误——Rate Limit 和 Token 计费误解

“429 Too Many Requests” 或 “Insufficient Quota” 这类错误同样常见。

首先，Claude Sonnet 4.5 是按 Token 计费的，你的余额对应的是 Token 额度。[云雾ai大模型聚合站](https://www.yunwuai.cc/)的计价规则是：**1 元人民币 = 1 美元 Token 额度**，按官方价格 1:1 换算。所以如果你调用了一个很长的上下文（超过 100K Token），费用会很高。

**避坑招数**：
1. 在请求体中显式设置 `max_tokens`，不要省略。默认值有时是 4096，但如果你不设置，模型可能“胡言乱语”或用尽你的额度。
2. 对于高并发场景，建议逐步增加请求量，不要一次性发起成千上万的并发。
3. 使用“限时特价分组”——[云雾ai大模型聚合站](https://www.yunwuai.cc/)针对 DeepSeek、Gemini 等模型有低至 0.6 倍费率的渠道，但 Claude Sonnet 4.5 通常在 1 倍费率（默认分组）或者 6~16 倍费率（官转/直连分组）。如果你的预算有限，先用默认分组做验证，稳定后再用官转。

## 第六大坑：响应内容乱码或截断——Model 输出不完整

代码跑通了，结果返回了一段奇怪的字符？或者输出到一半就停了？

**可能原因**：
- **编码问题**：确保你的代码、SDK 和目标应用都设置了 `utf-8` 编码。
- **最大 Token 限制**：检查你的 `max_tokens` 参数，是否设置得太小。Claude Sonnet 4.5 的输出能力非常强，如果你设置 `max_tokens=1024`，它输出完 1024 个 token 就停止了。
- **流式处理错误**：如果你用了 `stream=True`，但没有正确拼接返回的 chunks，会产生不完整的句子。请严格使用 OpenAI SDK 自带的流式处理方法（如 `for chunk in response`），不要手动拼接字符串。
