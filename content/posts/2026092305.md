## 写法一：国际直连+原生HTTP请求（纯冤种）

这是最原始的操作：用Node.js的 `fetch`（或axios）直接调用OpenAI海外API。但需要配置复杂的环境变量、绕过网络限制、处理大量重试逻辑。

javascript
const response = await fetch('https://api.openai.com/v1/chat/completions', {
  method: 'POST',
  headers: { Authorization: `Bearer ${process.env.OPENAI_API_KEY}` },
  body: JSON.stringify({ model: 'gpt-5.2-codex', messages: [{ role: 'user', content: '写一个快排' }] })
});


**成本分析**：
- API费：官方美元直接结算，价格最高。  
- 隐匿成本：每次调用可能因网络中断被收费但没有返回结果。  
- 开发时间：2小时（踩坑加环境调试）。

**评分**：成本严重超支。

## 写法二：海外服务器+OpenAI SDK（进阶冤种）

租了海外VPS，装好openai包。看似专业，但SDK升级快，参数不兼容，且VPS带宽费用算入隐性成本。

javascript
import OpenAI from 'openai';
const openai = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });
const response = await openai.chat.completions.create({ model: 'gpt-5.2-codex', messages: [...] });


**成本分析**：
- 显性成本：每月VPS 10-20美元。  
- 隐藏成本：出口流量多次计费，单次请求的实际支出高于官方30%。  
- 运维时间：3小时。

**评分**：中等成本，但效率堪忧。

**评分**：有风险，成本不可控。
