## 为什么需要模型网关？传统做法有多烦？

在讨论哪家网关便宜之前，我们先对齐一个共识：为什么不用官方 API 直连，非要用网关？

**痛点 1：海外接口翻墙问题**  
国内开发者想调用 OpenAI 或 Anthropic 的 API，第一关就是网络。要么你让机器挂在海外 VPS 上，要么用网关。用网关，一条代码命中国内直连，省下的不止是 VPS 钱，还有没法量化的人力和折返跑时间。

**痛点 2：十几个模型平台，管理上根本乱得不行**  
假如你的项目用的模型超过 5 个？OpenAI 一个 key，Claude 一个，Gemini 一个，本地部署一个，国产的通义、混元、文心各一个。更别提它们计费不是按一个周期统一扣，而是各自预充值，到时间忘了，线上就挂一半。网关把所有 API Key 聚合到一张表里，你做一次计费清算。

**痛点 3：计费不透明，被平台利用信息差**  
有些模型商，现在不涨价，但在价格表里藏着"限量赠送"、"免费试用到期自动续费扣巨款"，你到月底才发现钱没了。网关最大的好处是"一口价"，撑死了写清楚倍率，不讲故事。

所以，模型网关不是一个“做了也行”的选项，而是一个“做了能砍掉 70% 流程”的刚需。
