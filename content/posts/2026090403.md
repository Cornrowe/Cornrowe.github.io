## 第四步：常见报错及解决方案

新手最怕看到报错代码。我总结了几个你100%会遇到的问题，附上原因和秒杀方法：

### 1. **401 Authentication Error (Key无效)**
- **现象**：提示 `"You didn't provide an API key"` 或 `"Incorrect API key provided"`。
- **原因**：你复制错了Key，或者Key已过期/被删除。
- **解决**：检查Key是否正确复制，注意**不要有多余的空格或换行**。如果确认没错，去控制台重新生成一个新的Key试试。

### 2. **429 Rate Limit / Insufficient Quota (余额不足)**
- **现象**：提示 `"You have exceeded your current quota"` 或 `"Rate limit reached"`。
- **原因**：你的账户额度用完了，或者请求太频繁。
- **解决**：检查账户余额，低于0.1元最好别用。充个1块钱就能解决大部分问题。如果想降低频率，可以在代码里加 `time.sleep(1)` 控制一下。

### 3. **404 Model Not Found (模型不存在)**
- **现象**：提示 `"The model 'gpt-4-super' does not exist"`。
- **原因**：你打错了模型名，或者该模型在平台上不可用。
- **解决**：去云端API聚合站的“模型列表”页面，确认你用的模型名是否在支持列表中。常用的官方模型名如 `gpt-4o`、`gpt-4o-mini`、`claude-3-opus-20240229` 等，直接复制过去用。

### 4. **超时或连接错误 (Timeout/Connection Error)**
- **现象**：代码卡住很久，然后报 `requests.exceptions.Timeout` 或 `ConnectionError`。
- **原因**：网络不稳定，或者DNS解析出问题。
- **解决**：**绝大多数是网络问题**。检查你是否开了全局代理？如果开了，先关掉，因为云雾API聚合站是**国内直连**，不用代理。如果还是不行，重启路由器或者换一个网络环境（比如切手机热点）试试。
